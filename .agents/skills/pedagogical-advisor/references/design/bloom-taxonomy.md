# Bloom's Taxonomy (Revised)

Source: Anderson, L. W., Krathwohl, D. R., et al. (2001), *A Taxonomy
for Learning, Teaching, and Assessing: A Revision of Bloom's Taxonomy
of Educational Objectives*. Longman. (Revision of Bloom, 1956.)

Bloom's taxonomy is one of the oldest and most widely used frameworks
in educational design. It classifies cognitive processes at different
levels of complexity, giving instructors a vocabulary for specifying
what they want learners to do with knowledge.

The 2001 revision updated the original in important ways — it's the
version you should use when writing objectives or aligning
assessments to them.

## The cognitive process dimension

The revised taxonomy distinguishes six cognitive process categories,
ordered from simpler to more complex:

**1. Remember** — retrieve relevant knowledge from long-term memory.
- Recognize (identify)
- Recall (retrieve)
- Action verbs: define, list, identify, match, name, recall, recognize,
  state, label

**2. Understand** — construct meaning from instructional messages,
including oral, written, and graphic communication.
- Interpret (paraphrase, translate)
- Exemplify (give examples)
- Classify (categorize)
- Summarize (abstract, generalize)
- Infer (conclude, predict)
- Compare (contrast, match)
- Explain (construct causal models)
- Action verbs: describe, explain, summarize, paraphrase, classify,
  compare, contrast, predict, infer, estimate

**3. Apply** — carry out or use a procedure in a given situation.
- Execute (carry out routinely)
- Implement (use in novel contexts)
- Action verbs: apply, use, implement, carry out, execute, demonstrate,
  solve, show, calculate, modify, illustrate

**4. Analyze** — break material into its constituent parts and detect
how the parts relate.
- Differentiate (distinguish, focus, select)
- Organize (find coherence, integrate, outline, structure)
- Attribute (deconstruct motive or bias)
- Action verbs: analyze, differentiate, organize, attribute, outline,
  deconstruct, compare, contrast, examine, break down

**5. Evaluate** — make judgments based on criteria and standards.
- Check (coordinate, detect, monitor, test)
- Critique (judge)
- Action verbs: evaluate, critique, judge, justify, argue, defend,
  assess, support, decide, recommend

**6. Create** — put elements together to form a coherent whole;
reorganize elements into a new pattern or structure.
- Generate (hypothesize)
- Plan (design)
- Produce (construct)
- Action verbs: create, design, construct, generate, plan, produce,
  develop, formulate, compose, invent

### What changed from the original Bloom?

- Nouns became verbs. Original: Knowledge, Comprehension, Application,
  Analysis, Synthesis, Evaluation. Revised: Remember, Understand, Apply,
  Analyze, Evaluate, Create.
- The top two levels were swapped. "Synthesis" (original) became
  "Create" and moved to the top; "Evaluation" moved to second-highest.
- The cognitive process dimension became one of two dimensions (see
  below).

## The knowledge dimension

A contribution of the revision: the taxonomy now has *two* dimensions,
not one. The second dimension describes the *kind* of knowledge being
operated on.

**Factual knowledge** — basic elements learners must know to be
acquainted with a discipline. Terminology, specific details.

**Conceptual knowledge** — interrelationships between basic elements.
Classifications, principles, generalizations, theories, models,
structures.

**Procedural knowledge** — how to do something. Subject-specific
skills, algorithms, techniques, methods. Knowing when to use a
procedure is also procedural knowledge.

**Metacognitive knowledge** — knowledge of cognition in general, and
awareness of one's own cognition. Strategic knowledge, knowledge of
cognitive tasks, self-knowledge.

The two dimensions together form a grid:

|                   | Remember | Understand | Apply | Analyze | Evaluate | Create |
|-------------------|----------|------------|-------|---------|----------|--------|
| **Factual**       |          |            |       |         |          |        |
| **Conceptual**    |          |            |       |         |          |        |
| **Procedural**    |          |            |       |         |          |        |
| **Metacognitive** |          |            |       |         |          |        |

Every learning objective can be placed in one of these 24 cells —
though objectives often span multiple cells.

## Using the taxonomy in learning objectives

The standard use of Bloom is in writing learning objectives. A
well-written objective specifies:

- Who: the learner
- What: what they'll be able to do (action verb from the taxonomy)
- What kind of knowledge: from the knowledge dimension
- Conditions: under what circumstances (if relevant)
- Success criteria: what counts as meeting the objective

**Example of an objective at each Bloom level:**

- Remember (Factual): *Learners will be able to list the four
  chambers of the heart.*
- Understand (Conceptual): *Learners will be able to explain why
  blood flow through the heart follows the pattern it does.*
- Apply (Procedural): *Given a patient's EKG, learners will calculate
  the heart rate using the grid method.*
- Analyze (Conceptual): *Given a case of a specific cardiac
  abnormality, learners will identify which component of the heart's
  function is impaired.*
- Evaluate (Conceptual): *Learners will evaluate two proposed
  treatment plans for a patient with heart failure and argue which
  is more appropriate given the evidence.*
- Create (Procedural): *Learners will design a differential diagnosis
  algorithm for a patient presenting with chest pain.*

Notice how each successive level implies the previous ones. To
evaluate treatment plans, one has to analyze them, which requires
applying knowledge, which requires understanding, which requires
remembering.

## Using the taxonomy in assessment alignment

The taxonomy is equally valuable for checking whether assessments
match objectives. A common misalignment: objective is at the
*Understand* or *Apply* level; exam items test *Remember*.

When critiquing an assessment:

1. Identify the Bloom level of the stated objective.
2. Identify the Bloom level of each exam item.
3. Look for gaps — objectives with no items at the right level, or
   items at the wrong level.

**Common patterns**:
- Objectives ambitious (Analyze, Evaluate, Create) + items at
  Remember or Understand = hidden curriculum says "actually you just
  need to memorize."
- Objectives modest (Remember, Understand) + items at Apply or
  Analyze = students blindsided by what they're assessed on.
- All items at a single Bloom level = limited evidence of the range
  of understanding.

The fix: use a *test blueprint* that specifies, for each objective,
how many items at what Bloom level will test it. This forces
alignment before the exam is written, not after.

## Using the taxonomy in instruction

Different cognitive levels benefit from different instructional
activities:

**Remember and Understand**: exposure, organization, schema-building.
Lectures, readings, worked examples, retrieval practice. The
Dunlosky high-utility techniques (testing, spacing) are very
effective here.

**Apply**: practice with feedback. Worked examples followed by
independent practice. Scaffolded problem-solving.

**Analyze and Evaluate**: engagement with authentic complexity. Case
studies, comparative analysis, critique activities. Self-explanation
and discussion.

**Create**: extended projects, design tasks, open-ended investigation.
Scaffolded over time.

Most courses under-develop the higher levels because they're harder
to design for and harder to assess. Backward Design pushes back on
this by forcing the question of what the enduring understanding looks
like — which usually requires higher-level cognitive work.

## The limitations and caveats

Bloom's taxonomy is useful but has limits worth knowing:

- **It's a hierarchy, but not strictly.** The original presented the
  levels as strictly hierarchical. The revised version is more
  flexible — you can Evaluate without always Analyzing first, for
  example.
- **It focuses on cognition.** The original Bloom also had an
  affective and a psychomotor domain, less widely used. The taxonomy
  doesn't capture everything — motivation, collaboration, identity
  formation.
- **Level labels can be confusing.** "Understand" in Bloom means
  something specific (construct meaning); in common speech it means
  something broader. Being precise about which sense matters.
- **Levels are rough.** A task can involve multiple levels at once.
  Don't force an artificial classification.
- **Higher isn't always better.** A course about chemistry fundamentals
  should emphasize Remember, Understand, Apply. A graduate seminar
  should emphasize Analyze, Evaluate, Create. Matching the level to
  the learning goal is what matters, not pushing everyone to Create.

## Bloom vs. other taxonomies

Bloom is the most widely used but not the only useful taxonomy.
Others worth knowing:

**SOLO Taxonomy** (Biggs & Collis, 1982) — structures of the observed
learning outcome: prestructural, unistructural, multistructural,
relational, extended abstract. Focuses more on the structure of a
learner's response than on the cognitive process involved. Useful in
formative assessment.

**Webb's Depth of Knowledge** (1997) — four levels: recall,
skill/concept, strategic thinking, extended thinking. Widely used in
K-12 standards alignment. Similar spirit to Bloom but different
mapping.

**Marzano's New Taxonomy** (2001) — adds metacognitive and
self-system components to cognitive. Less widely used but
theoretically richer in some ways.

For pedagogical advisor purposes, Bloom (revised) is the default.
Others can be invoked when they fit better for a specific use.

## Critique checklist

When examining learning objectives and assessments:

1. Are objectives stated using Bloom-level action verbs, or are they
   vague ("understand X")?
2. Where on the taxonomy are the objectives? What's the range?
3. Do assessment items match the Bloom level of the objectives?
4. Is there a test blueprint or similar alignment document, or is
   alignment assumed?
5. Are the instructional activities building up to the Bloom level of
   the objectives, or are they at a lower level?
6. For the course as a whole, is there a reasonable mix of levels,
   or is everything at a single level?

## Common misuses

- **"Bloom's verbs" without Bloom's thinking.** Grabbing action verbs
  from a list without thinking about what kind of cognitive work the
  objective implies.
- **Treating higher as always better.** Pushing everything to
  "Create" is a misuse. Match the level to what the learner
  needs.
- **Using only the cognitive dimension.** Ignoring the knowledge
  dimension loses half the value. An Apply-Conceptual objective is
  different from an Apply-Procedural objective.
- **Writing objectives for the instructor's convenience.** "Students
  will be exposed to X" isn't an objective — no action verb, no
  measurable outcome.

## Summary

**Bloom's taxonomy (revised) is a shared vocabulary** for describing
what kind of cognitive work learners are being asked to do. It's
most useful in:

- Writing learning objectives with precise action verbs and knowledge
  types.
- Aligning assessments to objectives at the right cognitive level.
- Designing instructional activities that build toward the target
  level.
- Diagnosing where a course's objectives, instruction, or assessments
  don't match in their cognitive ambition.

It's a tool, not a prescription. Use it to be precise about what
you're designing for; don't use it as a checklist that forces
artificial levels where they don't fit.
